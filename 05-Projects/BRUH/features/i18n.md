# Internationalisation (i18n)

| Code | Language | Status |

| `ru` | Russian | Full |
| `tr` | Turkish | Full |

---

## Implementation

**File**: `src/lib/i18n.ts`

Language is stored in context and persisted to `nativeStorage` (Capacitor Preferences on native, localStorage on web).

```ts
import { useLanguage } from '@/contexts/LanguageContext';
const { t, language, setLanguage } = useLanguage();

// Use translation
<p>{t('create.step1Title')}</p>
```

---

## Query Key Rule for i18n

> [!warning] Translation-dependent query keys must be defined **inside components** (not at module level).
> Module-level keys are created once and won't react to language changes.

```ts
// ❌ WRONG — module-level, won't update when language changes
export const CHALLENGES_QUERY_KEY = ["active-challenges", language];

// ✅ CORRECT — inside component, reactive to language context
const { language } = useLanguage();
const queryKey = ["active-challenges", language];
```

This is an exception to the general rule of putting all query keys in `queryKeys.ts` — language-dependent keys must be inside the component.

## Translation Patterns

### Simple string
```tsx
<h2>{t("appearance.title")}</h2>
```

### String with variable (simple replace)
```tsx
t("login.notUser").replace("{user}", storedUsername ?? "")
```

### String with variable as JSX (bold/colored span)
```tsx
// "Type {username} to confirm" with bold username:
{t("delete.typeToConfirm").split("{username}")[0]}
<span className="font-bold text-destructive">{username}</span>
{t("delete.typeToConfirm").split("{username}")[1]}
```

### Reactive arrays — must be inside component body
Arrays whose items call `t()` must be declared **inside** the component function — not at module level — so they rebuild on language change:
```tsx
// ✅ Inside component
const TOGGLE_ITEMS = [{ label: t("notif.likes"), ... }];

// ❌ Module level — t() not in scope and won't react to language change
```
Affected: `TOGGLE_ITEMS`, `TYPE_META` (NotificationsSettings), `THEMES` (AppearanceSettings), `issueTypes` (HelpSettings), `filters` (MessageFilteringSettings).

## iOS vs Android Billing Strings

SubscriptionSettings renders billing note conditionally:
```tsx
{Capacitor.getPlatform() === "ios"
  ? t("sub.billingNoteIos")   // "Managed via App Store"
  : t("sub.billingNoteAndroid") // "Managed via Google Play"
}
```

## LanguageProvider in Tests

`LanguageProvider` must wrap components in test utils — without it, `t()` returns raw keys and text queries fail:

```tsx
// src/test/testUtils.tsx
function AllProviders({ children }) {
  return (
    <LanguageProvider>
      <QueryClientProvider client={queryClient}>{children}</QueryClientProvider>
    </LanguageProvider>
  );
}
```

## CreateScreen i18n

Step panels use single-line translations only — no extra description copy beneath the title (design decision):
- `create.step1Title` — single line
- `create.step2Title` — single line

## Adding a New Translation Key

1. Add to the `en` object in `src/lib/i18n.ts`
2. Add translations for `az`, `ru`, and `tr`
3. Use via `t('your.new.key')` in components
4. If the key is needed in a query key, define it **inside the component** (not module-level)
5. If it renders a variable with markup, use the JSX split pattern above
