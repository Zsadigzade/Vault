# Onboarding & first-run experience

---

## Value before friction

| Bad | Better |
|-----|--------|
| Sign-up wall before seeing content | Browse as guest / preview feed |
| Long tutorial carousel | **Contextual** tips on first real action |

---

## Permissions

- **Prime** before system dialog: “We need notifications to alert you to replies”
- Ask **when** the feature is used, not at launch (unless required)

See [[Notification UX Design]].

---

## Progressive profiling

- Collect **minimum** to start; enrich profile over time (badges, prompts after success)

---

## Empty states

- First-run empty ≠ error — show **sample** or **invite** CTA ([[Error State Design]])
