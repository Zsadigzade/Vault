# Growth loops

## TL;DR

- Loops beat funnels: output feeds back into input for compounding growth.
- Each loop: **input → action → output → reinvest into input**.
- K-factor > 1 = viral; < 1 = subsidized. Know which you have.

## Why loops > funnels

A funnel is linear (spend → acquire → done). A loop is circular — the output of one cycle becomes the input for the next. Loops compound; funnels only scale with spend.

---

## The 4 core growth loop types

### 1. Viral / Sharing loops
User action creates exposure → new users join

```
User creates content / sends invite
  → Recipient sees product
  → Recipient joins
  → Recipient creates content / invites
```

**Key metric:** K-factor = (invites sent per user) × (conversion rate)
- K > 1: viral, growth accelerates
- K < 1: viral assists growth but doesn't sustain it alone

**Examples:** WhatsApp (invite), TikTok (share to Instagram), Dropbox (referral)

### 2. Content / SEO loops
User-generated or created content drives organic discovery

```
User or product creates content
  → Indexed by search / surfaced by algorithm
  → New user discovers and joins
  → New user creates more content
```

**Key metric:** Indexed pages per cohort × organic conversion rate
**Examples:** Reddit, Quora, YouTube, any UGC platform

### 3. Paid acquisition + retention loops
Paid acquisition works only when LTV > CAC × payback window

```
Spend on ads
  → Acquire user
  → User generates revenue (subscription / transaction)
  → Revenue funds more ad spend
```

**Key metric:** LTV:CAC ratio (healthy = 3:1+); payback period (healthy = <12 months)
**Compounds when:** Revenue per user grows (expansion, upsell) faster than CAC

### 4. Product-led / engagement loops
Using the product drives more usage which drives more value

```
User engages with feature
  → More data / connections / network value accumulates
  → Product becomes more valuable for that user
  → User engages more (and invites others)
```

**Key metric:** D30/D90 retention; daily active rate
**Examples:** Spotify Wrapped (data flywheel), LinkedIn (connections), Figma (multiplayer)

## Designing a loop

1. **Name the repeatable unit** — what one user action triggers the loop? (post, invite, transaction, review)
2. **Draw the cycle** — input → mechanism → output → back to input
3. **Find the rate limiter** — what's slowing the loop? (friction, incentive, reach)
4. **Pick the health metric** — K-factor, repeat rate, organic acquisition share, LTV:CAC

## Loop health questions

| Question | What it reveals |
|---------|----------------|
| What % of new users come from the loop (vs paid)? | Loop strength vs subsidy |
| Is K-factor trending up or down? | Viral loop health |
| What's the payback period on paid? | Whether paid loop is self-funding |
| Are D30 retention cohorts stable or eroding? | Engagement loop health |
